## Jam vs Studio Architecture: The Critical Question

### TL;DR: Use Studio Mode for Onboarding

**Studio Mode** (`/studio`):
- ‚úÖ 6 modules (Drum, Piano, Guitar, Chord Arranger, Visualizer, 3D Sequencer)
- ‚úÖ Drag-and-drop module system
- ‚úÖ Module routing (send MIDI between modules)
- ‚úÖ Responsive design (mobile ‚Üí desktop)
- ‚úÖ Production-ready UX
- ‚úÖ No backend dependency

**Jam Mode** (`/jam`):
- ‚úÖ Real-time collaboration (Supabase)
- ‚úÖ Room codes (6-char)
- ‚úÖ Presence tracking
- ‚ö†Ô∏è Only 1 module synced (Drum Machine)
- ‚ö†Ô∏è Simpler UX (tab-based)

**Verdict**: **Studio is the primary onboarding entry point**, Jam is the "collaboration upsell" after tutorial completion.

### Tutorial Routing Strategy

```
User clicks: jam.audiolux.app?v=m (Musician)
  ‚Üì
Loads: /studio with Drum Machine pre-loaded
  ‚Üì
Tutorial: "Create kick pattern ‚Üí Add snare ‚Üí Press play"
  ‚Üì
Completion: "Share with friends" ‚Üí Creates Jam Session ‚Üí Redirects to /jam
```

**Why this works**:
- Studio has all modules for persona-specific tutorials
- Jam is the viral loop (collaboration = sharing)
- Clean separation: Learning (Studio) ‚Üí Sharing (Jam)
